Detailed Description

template<class KINEMATICS, bool FLOATING_BASE, size_t STATE_DIM, size_t CONTROL_DIM>
class ct::rbd::TermTaskspacePoseCG< KINEMATICS, FLOATING_BASE, STATE_DIM, CONTROL_DIM >

A costfunction term that defines a cost on a task-space pose, uses code-generation internally and allows to have end-effector position and orientation as parameters in auto-diff.

This term is desigend as term for "ct::optcon::CostFunctionAnalytic", but uses AD internally.

Warning
this term implements a flag which allows to rid of the evaluation of the control-derivatives (they are zero) and just set them to zero. The reason for this flag is that it is the only way to avoid multiple evaluations of the Jacobians and Hessians. If the jacobians and hessians were member variables, the overhead of evaluating them multiple times could be avoided. NOTE: setting this flag is not a good idea for purely kinematic systems!
Template Parameters
KINEMATICSkinematics of the system
FLOATING_BASEtrue if system is a floating base robot
STATE_DIMstate dimensionality of the system
CONTROL_DIMcontrol dimensionality of the system
